# `compass onramp buy` — hand-written orchestration (activation guide)

> **Status:** this package is intentionally **doc-only** in the committed tree.
> The Go command below references the **generated** `s.Onramp` SDK accessor, which
> only exists **after** Speakeasy regenerates `cli-sdk/` from the Task 02
> `/v2/onramp/*` routes. Committing the `.go` files before that regen would break
> `go build ./...` (the symbols don't exist yet). Apply the steps here in the same
> change as the first regen. Authored as part of
> `docs/plans/2026-06-23-halliday-onramp-integration/` Task 05.

## Why a separate `onrampbuy` package (not `internal/cli/onramp/`)

Unlike `internal/cli/risk/` (which has no OpenAPI route, so Speakeasy never touches
its directory), the on-ramp routes DO generate a package at
`internal/cli/onramp/` (quote/create/status). A hand-written file in that same dir
would collide with / be overwritten by regen. So the preserved `buy` command lives
in its **own** package `internal/cli/onrampbuy/` and **attaches** itself to the
generated `onramp` parent command at wiring time.

## What auto-surfaces for free (no code)

- **MCP:** `mcp_server.py`'s `INCLUDE_PATH_PREFIXES = ("/v2/",)` already matches
  `/v2/onramp/*` → `v2_onramp_quote` / `v2_onramp_create` / `v2_onramp_status`
  tools appear on the next API restart. **No MCP change.** The agent guidance
  (present `checkout_url`, poll `v2_onramp_status`) comes from the Task 02 route
  docstrings + the `user_instructions` response field.
- **CLI quote/create/status:** generated by Speakeasy because
  `.speakeasy/cli-overlay.yaml` does **not** remove `/v2/onramp/*` (it only adds
  the `x-speakeasy-name-override` entries — already committed in this change — so
  the commands are `compass onramp quote|create|status`, not `onramp-quote`).

## Activation steps (do alongside the first regen)

1. Run the regen: `make run_api && make update_openapi_json && make stop_api`,
   then `speakeasy run -t compass-cli`. Confirm `openapi.json` has
   `/v2/onramp/{quote,create,status}` and that `cli-sdk/internal/sdk/onramp.go`
   (`OnrampQuote/OnrampCreate/OnrampStatus`) + `internal/cli/onramp/` were
   generated and `s.Onramp` exists on `CompassCLI`.
2. Create `cli-sdk/internal/cli/onrampbuy/onrampbuy.go` and `browser.go` from the
   reference below (NO `// Code generated by Speakeasy` header → regen ignores
   them). Adjust the generated request/response type + field names to whatever
   `speakeasy run` actually emitted (e.g. `components.OnrampCreateRequest`,
   `res.OnrampCreateResponse.CheckoutURL` / `.PaymentID` / `.UserInstructions`).
3. Add `github.com/pkg/browser` to `cli-sdk/go.mod` (`go get github.com/pkg/browser`).
4. Re-inject the wiring into the **generated** `internal/cli/root.go` via the
   release workflow, mirroring the existing risk block in
   `.github/workflows/generate_combined_spec_and_sdks.yaml` ("Patch root.go"):

   ```bash
   # import (alphabetical anchor; onrampbuy sorts after onramp)
   if ! grep -q '"github.com/CompassLabs/cli/internal/cli/onrampbuy"' "$ROOT"; then
     sed -i 's|"github.com/CompassLabs/cli/internal/cli/risk"|"github.com/CompassLabs/cli/internal/cli/onrampbuy"\n\t"github.com/CompassLabs/cli/internal/cli/risk"|' "$ROOT"
   fi
   # registration — AFTER the generated onramp root is wired, so `buy` attaches to it
   if ! grep -q 'onrampbuy.InitOnrampBuy(rootCmd)' "$ROOT"; then
     sed -i 's|\tinitExploreCmd(rootCmd)|\tinitExploreCmd(rootCmd)\n\n\tif err := onrampbuy.InitOnrampBuy(rootCmd); err != nil {\n\t\treturn nil, fmt.Errorf("init onramp buy: %w", err)\n\t}|' "$ROOT"
   fi
   ```
5. `cd cli-sdk && go build ./... && go test ./...`. Smoke: `compass onramp buy
   --owner 0x… --amount 100 --dry-run` (opens nothing); piped (`| cat`) prints the
   URL + instructions with no spinner garble.

## Reference implementation

`InitOnrampBuy` find-or-creates the `onramp` parent (the generated package usually
creates it) and attaches `buy`. The RunE mirrors `internal/cli/earn/earnmanage.go`
(`client.NewClient` → `output.PrepareCallOpts` → `client.IsDryRun` → generated SDK
call → `output.Result`/`output.Error`), then adds the browser-handoff + poll:

```go
// internal/cli/onrampbuy/onrampbuy.go  (hand-written; no Speakeasy header)
package onrampbuy

import (
	"context"; "fmt"; "os"; "time"
	"github.com/CompassLabs/cli/internal/client"
	"github.com/CompassLabs/cli/internal/output"
	"github.com/CompassLabs/cli/internal/sdk/models/components"
	"github.com/CompassLabs/cli/internal/sdk/models/operations"
	"github.com/spf13/cobra"
	"golang.org/x/term"
)

// InitOnrampBuy attaches `buy` to the generated `onramp` command (find-or-create).
func InitOnrampBuy(parent *cobra.Command) error {
	onramp := findOrCreateOnramp(parent)
	buy := &cobra.Command{
		Use:   "buy",
		Short: "Buy crypto with a card — opens a hosted checkout and polls until delivered",
		Long:  "Quote → create a Halliday on-ramp → open the hosted card/KYC checkout in your browser → poll status until USDC is delivered. The checkout (card entry + KYC) only renders in a browser, so this command hands off to one (like `gh auth login`).",
		RunE:  runBuy,
	}
	buy.Flags().String("owner", "", "Destination EVM wallet address (receives USDC). [required]")
	buy.Flags().String("amount", "", "Fiat/USDC amount to buy. [required]")
	buy.Flags().String("chain", "ethereum", "Destination chain (USDC-on-Ethereum only for now).")
	_ = buy.MarkFlagRequired("owner")
	_ = buy.MarkFlagRequired("amount")
	onramp.AddCommand(buy)
	return nil
}

func findOrCreateOnramp(parent *cobra.Command) *cobra.Command {
	for _, c := range parent.Commands() {
		if c.Name() == "onramp" {
			return c
		}
	}
	c := &cobra.Command{Use: "onramp", Short: "Fiat on-ramp"}
	parent.AddCommand(c)
	return c
}

func runBuy(cmd *cobra.Command, _ []string) error {
	owner, _ := cmd.Flags().GetString("owner")
	amount, _ := cmd.Flags().GetString("amount")
	chain, _ := cmd.Flags().GetString("chain")

	s, err := client.NewClient(cmd)
	if err != nil {
		return err
	}
	opts, err := output.PrepareCallOpts(cmd)
	if err != nil {
		return err
	}
	dry := client.IsDryRun(cmd)
	if dry {
		opts = append(opts, operations.WithSkipDeserialization())
	}

	// 1) quote (confirm pricing) — field/type names per the generated SDK.
	quoteReq := components.OnrampQuoteRequest{Owner: owner, Amount: amount, Chain: chain}
	qres, err := s.Onramp.OnrampQuote(cmd.Context(), quoteReq, opts...)
	if err != nil {
		return output.Error(cmd, err)
	}
	_ = output.Result(cmd, qres)

	// 2) create — read checkout_url / payment_id / user_instructions off the response.
	createReq := components.OnrampCreateRequest{Owner: owner, Amount: amount, Chain: chain}
	cres, err := s.Onramp.OnrampCreate(cmd.Context(), createReq, opts...)
	if err != nil {
		return output.Error(cmd, err)
	}
	if dry {
		return output.Result(cmd, cres)
	}
	checkoutURL := cres.OnrampCreateResponse.CheckoutURL
	paymentID := cres.OnrampCreateResponse.PaymentID
	instructions := cres.OnrampCreateResponse.UserInstructions

	// 3) ALWAYS print the URL + instructions first (never lose it), then try to open.
	fmt.Fprintf(os.Stderr, "%s\nOpening your card checkout in the browser. If it doesn't open, visit:\n  %s\n", instructions, checkoutURL)
	interactive := term.IsTerminal(int(os.Stdout.Fd())) && !output.IsAgentMode()
	if interactive {
		_ = openInBrowser(checkoutURL) // best-effort; URL already printed
	}

	// 4) poll status until terminal (delivered/failed/expired) or ~15m timeout.
	ctx, cancel := context.WithTimeout(cmd.Context(), 15*time.Minute)
	defer cancel()
	for {
		select {
		case <-ctx.Done():
			fmt.Fprintf(os.Stderr, "Still pending — re-check with `compass onramp status --payment-id %s`\n", paymentID)
			return nil
		default:
		}
		sres, err := s.Onramp.OnrampStatus(ctx, operations.OnrampStatusRequest{PaymentID: paymentID}, opts...)
		if err != nil {
			return output.Error(cmd, err)
		}
		st := sres.OnrampStatusResponse.Status // e.g. "delivered" | "failed" | "expired"
		if st == "delivered" || st == "failed" || st == "expired" {
			return output.Result(cmd, sres)
		}
		if interactive {
			fmt.Fprintf(os.Stderr, "\r⠋ waiting for delivery… (%s)", st)
		}
		time.Sleep(3 * time.Second)
	}
}
```
